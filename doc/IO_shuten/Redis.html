Implementation of the Redis storage

Two backends are possible: simple key-value and pub-sub messaging

KeyValue has different node types. Incremental single message store: each write will create a new key. Message collection store: all writes go to a single key. Good for events or logs.

PubSub has different node types: publisher and subscriber. In future maybe a combined publisher-subscriber type possible, but in general less needed. Good for live event messaging.
